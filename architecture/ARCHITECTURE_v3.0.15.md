### 2.1 令牌桶限流架构

```
┌─────────────────────────────────────────────────────────────────────────────────────┐
│                         令牌桶限流架构 (Token Bucket Rate Limiting)                   │
├─────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                      │
│  请求流程                                                                             │
│     │                                                                                │
│     ▼                                                                                │
│  ┌─────────────────────────────────────────────────────────────────────────────┐    │
│  │                     Rate Limit Middleware                                      │    │
│  │  ┌─────────────────┐                                                          │    │
│  │  │  Scope Builder  │  提取限流维度:                                             │    │
│  │  │                 │  - user_id (用户)                                        │    │
│  │  │                 │  - org_id (组织)                                         │    │
│  │  │                 │  - api_key (API Key)                                     │    │
│  │  │                 │  - ip_address (IP)                                       │    │
│  │  │                 │  - endpoint (端点)                                       │    │
│  │  └────────┬────────┘                                                          │    │
│  │           ▼                                                                   │    │
│  │  ┌─────────────────────────────────────────────────────────────────────┐   │    │
│  │  │                    Redis Lua Script (原子操作)                         │   │    │
│  │  │                                                                     │   │    │
│  │  │  local tokens = redis.call('GET', bucket_key .. ':tokens') or bucket │   │    │
│  │  │  local last_refill = redis.call('GET', bucket_key .. ':refill')     │   │    │
│  │  │                                                                     │   │    │
│  │  │  -- 计算令牌补充                                                     │   │    │
│  │  │  elapsed = now - last_refill                                         │   │    │
│  │  │  refill = elapsed * refill_rate                                       │   │    │
│  │  │  tokens = min(capacity, tokens + refill)                              │   │    │
│  │  │                                                                     │   │    │
│  │  │  -- 尝试消费令牌                                                      │   │    │
│  │  │  if tokens >= 1 then                                                  │   │    │
│  │  │    tokens = tokens - 1                                                │   │    │
│  │  │    return {1, tokens, capacity}  -- 允许                              │   │    │
│  │  │  else                                                                 │   │    │
│  │  │    return {0, tokens, reset_time}  -- 拒绝                             │   │    │
│  │  │  end                                                                  │   │    │
│  │  │                                                                     │   │    │
│  │  └─────────────────────────────────────────────────────────────────────┘   │    │
│  │           │                                                                   │    │
│  │           ▼                                                                   │    │
│  │  ┌─────────────────────────────────────────────────────────────────────┐   │    │
│  │  │                    限流决策                                             │   │    │
│  │  │                                                                     │   │    │
│  │  │   允许 ──────▶ 执行请求 ──────▶ 记录指标 ──────▶ 更新 Redis            │   │    │
│  │  │                      │                                               │   │    │
│  │  │   拒绝 ──────▶ 返回 429 ──────▶ 记录拒绝 ──────▶ 发送告警(阈值时)     │   │    │
│  │  │                                                                     │   │    │
│  │  └─────────────────────────────────────────────────────────────────────┘   │    │
│  └─────────────────────────────────────────────────────────────────────────────┘    │
│                                                                                      │
│  限流层级                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────────────┐    │
│  │                                                                              │    │
│  │   Tier 1: IP 限流 (防止 DDoS, 无需认证)                                      │    │
│  │   Tier 2: 用户限流 (认证用户, 基于 user_id)                                  │    │
│  │   Tier 3: 组织限流 (共享配额, 基于 org_id)                                   │    │
│  │   Tier 4: API Key 限流 (第三方集成, 基于 api_key)                            │    │
│  │   Tier 5: 端点限流 (特殊敏感端点)                                           │    │
│  │                                                                              │    │
│  │   所有层级同时生效, 任一层超限即拒绝                                          │    │
│  │                                                                              │    │
│  └─────────────────────────────────────────────────────────────────────────────┘    │
│                                                                                      │
└─────────────────────────────────────────────────────────────────────────────────────┘
```
